# Reglas de Trabajo - Baroid PWA

1. **Reglas de Autonomía y Validación**

   - **Validación Local:** "Cada vez que hagas un cambio en el código, levanta el servidor local (ej. npm run dev) para verificar que no haya errores de compilación. No uses Docker para esto."
   
   - **Ejecución Controlada:** "NO ejecutes comandos automáticamente sin permiso explícito, salvo para levantar el servidor de desarrollo local. Para instalaciones o cambios mayores, solicita confirmación."

   - **Verificación UI:** "Al verificar cambios visuales, confía en la estructura del código y haz pruebas lógicas. NO es necesario realizar capturas de pantalla a menos que se solicite explícitamente."

2. **Manejo de Rutas y Terminal (Windows)**

   - **Rutas Compatibles:** "Utiliza siempre rutas compatibles con Windows (backward slashes `\`). Prefiere PowerShell."
   
   - **Gestión de Errores:** "Si un comando falla, intenta analizar el error y proponer una solución, pero NO ejecutes correcciones automáticas complejas sin avisar."

3. **Estilo de Workflows**

   - **Scripts:** "Para automatizaciones, usa scripts de Node.js simples y modulares."
